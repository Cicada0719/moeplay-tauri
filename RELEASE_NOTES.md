# 萌游 MoeGame v0.19.3

## 本地 Galgame 启动修复

- 修复 0.19.2 在 Windows 上打开本地 Galgame 后萌游闪退的问题。
- 游戏进程监控不再依赖同步命令线程必须处于 Tokio runtime，改为独立命名线程等待子进程退出，避免游戏成功启动后触发不可恢复的后端 panic。
- 监控线程初始化失败时会安全清理运行记录和游玩会话并返回可读错误，不再导致整个应用退出。

## 小说多书源与下载

- 小说模块现在提供 7 个入口：全部书源、笔趣阁、Internet Archive、Open Library、Standard Ebooks、Project Gutenberg 与中文维基文库。
- 新增 Internet Archive 公共文本/扫描本搜索、正文阅读及上游 EPUB/PDF 下载映射。
- 新增 Open Library 开放书目搜索，仅选择带公开扫描的记录，并读取对应 Internet Archive 纯文本。
- 新增 Standard Ebooks 精校公版书搜索与兼容 EPUB 下载。
- 笔趣阁支持作品搜索、分页目录与分页章节正文；单一书源请求或页面结构异常时会显示明确错误，不再静默返回空结果。
- 应用只读取无需登录即可访问的公开页面和开放接口，不绕过付费墙、登录或访问控制；下载按钮仅在上游明确提供文件时显示。

## 界面与交互

- 漫画宽屏页面加入纯 CSS 响应式背景、渐变、纹理与内容层次，填充两侧空白；窄屏、高对比和减少动态效果模式会自动降级。
- 专注模式改为单一临时状态，不再按栏目持久化隐藏控件；切换栏目会自动退出，避免返回页面后 UI 莫名缺失。
- 统一“专注模式 / 退出专注”按钮和手柄提示，改善高缩放、窄窗口与低高度窗口下的恢复入口和横向溢出。

## 验证

- Rust 小说解析测试 11 项通过、1 项网络实测保留为 ignored；Galgame 无 Tokio runtime 回归测试通过。
- Svelte / TypeScript 检查 0 错误、0 警告；前端单元测试 656 项通过、1 项跳过。
- 漫画、专注/缩放、手柄导航定向 Playwright 覆盖通过。
- 使用本机 Kirikiri Galgame《逐光柠檬协奏曲》实测：从发布构建成功打开启动器，MoePlay 在游戏进程运行及退出后保持响应，未复现 0.19.2 闪退。
