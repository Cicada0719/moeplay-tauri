### 正文

大家好，我最近一直在做一个 Windows 上的开源项目，叫 **萌游 MoeGame**。

最开始的想法很简单：Steam、Epic、本地游戏、模拟器、番剧和漫画都散落在不同软件里，真正想找点东西玩的时候，反而要先在一堆窗口和文件夹之间来回翻。

所以我试着做了一个统一入口——它既能在桌面上整理游戏资料，也能接到电视上，用手柄进入一套完全独立的大屏界面。

这次我重点做的是 **大屏滚动剧场**：

- 游戏背景铺满整块屏幕，标题和启动操作放在左下角，不挡主画面；
- 底部是可以用左摇杆或十字键手动切换的封面轮，不会像轮播图一样自己乱跳；
- 普通游戏库、详情、搜索、媒体展厅和弹窗都补了手柄焦点；
- 可以一键进入“专注布局”，把暂时用不到的侧栏和控件收起来。

除了大屏模式，目前还有这些功能：

- 导入 Steam、Epic、本地游戏和模拟器 ROM；
- 从 Bangumi、VNDB、DLSite、Steam、PCGamingWiki 等来源补全封面和资料；
- 管理标签、合集、评分、收藏、笔记、启动参数和日语环境启动；
- 记录游玩时长、最近启动、继续游玩和周期统计；
- 番剧多源搜索、换源、HLS / 网页 / 外部播放器兜底、弹幕和播放进度；
- 漫画阅读，可接 Komga、Kavita 和本地来源；
- Project Gutenberg、中文维基文库小说阅读；
- 下载任务、来源健康检查、备份、诊断、缓存清理和签名自动更新。

技术上使用 **Tauri 2 + Rust + Svelte 5 + SQLite**。核心资料默认保存在本机，项目采用 MIT License，源码、构建流程和发布校验都在 GitHub 上公开。

目前只发布 **Windows 10/11 x64** 版本，提供安装版和便携版，暂时没有 Android APK。

如果你也喜欢整理游戏库、折腾客厅电脑、用手柄逛收藏，欢迎试试看。任何界面建议、来源兼容问题和手柄操作反馈，都可以直接在 GitHub Issue 里告诉我。

### 剪辑与表达建议

- 第一秒直接展示最有辨识度的大屏画面，不要先放 Logo 动画。
- 切换封面时保留真实手柄输入和焦点动画，让观众看懂“不是概念图”。
- 前 15 秒只讲用户能获得什么，不先解释技术栈。
- 技术栈放在后半段或评论区，既能吸引普通用户，也方便开发者判断项目质量。
- 画面中避免出现来源账号、API Key、本地绝对路径和私人游戏记录。
- 如果展示第三方作品封面，优先选取自己拥有且适合公开展示的内容。
