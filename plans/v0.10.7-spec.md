# v0.10.7 详细规格：全局键盘快捷键系统

## 1. 快捷键映射

```ts
export interface ShortcutDefinition {
  id: string;           // 唯一标识
  keys: string;         // 展示文本，如 "Ctrl + K"
  trigger: {            // @svelte-put/shortcut 触发器
    key: string;
    modifier?: "ctrl" | "meta" | "alt" | "shift" | ("ctrl" | "meta" | "alt" | "shift")[] | false;
    code?: string;
  };
  description: string;  // 帮助文案
  scope: "global" | "home"; // global 全局生效，home 仅在游戏库页生效
  action: () => void;   // 执行函数
}
```

### 1.1 默认快捷键

| 按键 | 作用 | 范围 |
|------|------|------|
| `1` | 切换到「游戏库」 | global |
| `2` | 切换到「继续」 | global |
| `3` | 切换到「番剧」 | global |
| `4` | 切换到「漫画」 | global |
| `5` | 打开/关闭「工具抽屉」 | global |
| `Escape` | 返回首页（保持现有行为） | global |
| `?` | 打开快捷键帮助 | global |
| `/` | 聚焦游戏库搜索框 | home |
| `Ctrl/Cmd + K` | 聚焦游戏库搜索框 | home |

## 2. 核心模块（src/lib/shortcuts.ts）

### 2.1 视图映射

```ts
export const DOCK_SHORTCUT_VIEWS: { index: number; view: string; label: string }[]
```

对应 DOCK_ITEMS 前 5 项：home / continue / anime / comic / __tools。

### 2.2 触发器构建

```ts
export function buildDockShortcuts(
  navigate: (view: string) => void,
  toggleTools: () => void
): ShortcutDefinition[]
```

生成数字 1-5 的触发器与回调。

### 2.3 帮助数据

```ts
export function buildShortcutsHelp(
  navigate: (view: string) => void,
  toggleTools: () => void,
  focusSearch: () => void,
  toggleHelp: () => void
): ShortcutDefinition[]
```

返回完整快捷键列表，用于帮助浮层渲染。

### 2.4 视图解析

```ts
export function resolveDockIndexByKey(key: string): number | null
```

解析 `Digit1`~`Digit5` / `Numpad1`~`Numpad5` 到 0-based 索引。

## 3. UI 规格

### 3.1 App.svelte 注册

- 在 `<svelte:window>` 或顶层 div 上通过 `use:shortcut` 注册全局触发器。
- 由于 `use:shortcut` 是 Svelte action，需绑定到真实 DOM 元素。App.svelte 顶层是 `<div class="app-container">`，可在此注册全局快捷键。
- 保持现有 `onKeydown` Escape 处理，但将 `?` 和帮助浮层逻辑抽出。

### 3.2 快捷键帮助浮层

`ShortcutHelp.svelte`：

- 固定居中卡片，宽度 `min(480px, calc(100vw - 40px))`。
- 标题「快捷键」。
- 按范围分组：全局、游戏库。
- 每项左侧显示 `<kbd>` 样式按键，右侧显示说明。
- 底部「关闭」按钮，Escape 在浮层打开时优先关闭浮层。

### 3.3 Dock 数字提示

`SystemDock.svelte`：

- 在 Dock 项标签附近显示小数字角标 `1`~`5`（仅前 5 项）。
- 样式：8px 圆角，半透明背景，10px 字号。
- 帮助用户发现快捷键。

### 3.4 搜索聚焦

`SwitchHome.svelte`：

- 暴露 `focusSearch()` 方法或在 store 中设置 `_searchFocused` 标记。
- 当快捷键触发时，若当前视图不是 home，先切换到 home 再聚焦搜索。
- 使用 `bind:this` 获取 input 元素并调用 `.focus()`、`.select()`。

## 4. 测试规格

1. `resolveDockIndexByKey`：解析 Digit/Numpad 1-5，忽略其他键。

3. `buildShortcutsHelp`：包含核心快捷键且描述非空。
