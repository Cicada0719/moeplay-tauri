# v0.10.11 详细规格：URL hash 路由持久化

## 1. 路由 Store（`src/lib/stores/router.svelte.ts`）

```ts
export type RouteParams = { gameId?: string };
export interface AppRoute {
  view: string;
  params: RouteParams;
}

export const KNOWN_VIEWS: string[];

export function isKnownView(view: string): boolean;
export function parseHash(hash: string): AppRoute;
export function buildHash(view: string, params?: RouteParams): string;
export function applyHash(): void;
export function initRouter(): void;
```

### 1.1 已知视图白名单

从 `DOCK_ITEMS` 与 `TOOL_ITEMS` 收集所有合法 `view`，并额外加入：

- `home`
- `game-detail`

过滤掉内部视图 `__tools`、`__bigpicture`（不持久化）。

### 1.2 hash 格式

- 普通视图：`#home`、`#settings`、`#stats`
- 游戏详情：`#game-detail?id=<gameId>`
- 空或非法 hash 视为 `home`。

### 1.3 `applyHash`

- 读取 `window.location.hash`。
- 若视图非法，静默回退 `home`。
- 设置 `uiStore.currentView = route.view`。
- 若 `route.view === "game-detail"` 且 `route.params.gameId` 存在，调用 `gameStore.selectGame(gameId)`。

### 1.4 双向同步

- `$effect` 监听 `uiStore.currentView` 与 `gameStore.selectedId`。
- 当视图或选中游戏变化时，调用 `buildHash` 更新 `window.location.hash`。
- 仅在非初始化阶段触发，避免与 `applyHash` 互相覆盖。

### 1.5 历史导航

- 监听 `hashchange` 事件，在 hash 因浏览器前进/后退变化时调用 `applyHash`。
- 监听 `popstate` 作为兜底。

### 1.6 `initRouter`

- 在 `App.svelte` 的 `onMount` 中调用。
- 先触发一次 `applyHash()` 恢复状态，再启动同步 effect 与事件监听。

## 2. UI Store 调整

- 移除 `let _currentView = $state(new URLSearchParams(window.location.search).get("view") || "home");`
- 改为 `let _currentView = $state("home");`
- 保留其余逻辑不变。

## 3. App.svelte 调整

- 导入 `initRouter`。
- 在 `onMount` 的 `booted` 逻辑之后调用 `initRouter()`。
- 原有的 `pickDock`、`pickTool`、`goHome` 等函数无需改动，因为 router 的 effect 会自动同步 hash。

## 4. 测试规格

`src/lib/stores/router.test.ts` 覆盖：

1. `isKnownView`：识别所有合法视图，拒绝非法与内部视图。
2. `parseHash`：解析普通 hash、带参数的 hash、空 hash、非法 hash。
3. `buildHash`：生成普通 hash 与游戏详情 hash。
4. 非法视图回退到 `home`。
