# v0.12.2 Stage 2 — Unified Task Center SPEC

## 1. Objective

将现有持久化 `BackgroundJob`/`TaskQueue`、下载任务、AI 任务及各页面局部任务展示统一为应用级 Task Center。用户应能在一个入口查看、筛选、取消、暂停、恢复、重试和清理任务，并在应用重启后看到恢复状态。

## 2. Existing Foundation

Backend:
- `src-tauri/src/task_queue.rs`
- `src-tauri/src/repositories/jobs.rs`
- `src-tauri/src/commands/tasks.rs`

Frontend:
- `src/lib/features/jobs/{contracts,api,store}.ts`
- `src/lib/features/jobs/JobsPanel.svelte`
- `src/lib/components/DownloadPage.svelte`

Current limitation:
- Jobs UI 主要嵌入下载体验。
- 下载/AI/通用任务仍有局部投影和不同状态表达。
- 缺少全局入口、统一过滤、失败分类、恢复动作和任务详情。

## 3. Domain Contract

Task Center public projection:

```ts
type JobKind =
  | "download"
  | "import"
  | "scrape"
  | "provider_verify"
  | "ai"
  | "backup"
  | "restore"
  | "diagnostics"
  | "update"
  | "generic";

type JobStatus =
  | "queued"
  | "running"
  | "paused"
  | "succeeded"
  | "failed"
  | "cancelled";

interface TaskCenterJob {
  id: string;
  kind: JobKind;
  title: string;
  status: JobStatus;
  progress: number; // 0..1
  message?: string;
  errorKind?: string;
  retryable: boolean;
  resumable: boolean;
  cancellable: boolean;
  recovered: boolean;
  createdAt: string;
  updatedAt: string;
  idempotencyKey?: string;
  source?: { area: string; entityId?: string; label?: string };
}
```

兼容要求：Rust serde 字段与 TS projection 必须保持已有命名风格，必要时通过 mapper 兼容旧 DTO，不得一次性破坏现有 DownloadPage。

## 4. Work Packages

### S2-A Backend task control plane

Owned files:
- `src-tauri/src/task_queue.rs`
- `src-tauri/src/repositories/jobs.rs`
- `src-tauri/src/commands/tasks.rs`
- 对应 Rust tests，可新增 `src-tauri/tests/task_center_contract.rs`

Requirements:
1. 完善任务 kind/status/action capability 投影。
2. 列表支持可选过滤：status、kind、limit。
3. retry/pause/resume 通过明确命令边界实现；不支持的 kind 返回结构化错误。
4. cancel 必须保持真实取消句柄语义。
5. 重启恢复时 running 任务不得虚假继续；应映射为 paused/queued/failed 中明确可恢复状态。
6. clear finished 只清理 terminal jobs。
7. 保持旧 `get_tasks` 等命令兼容，或提供版本化命令并同步命令契约。

### S2-B Frontend store/domain

Owned files:
- `src/lib/features/jobs/contracts.ts`
- `src/lib/features/jobs/api.ts`
- `src/lib/features/jobs/store.ts`
- `src/lib/features/jobs/store.test.ts`
- 可新增 jobs 目录下测试/helper。

Requirements:
1. 建立单一 Task Center projection。
2. 支持 load/refresh/filter、乐观操作与失败回滚。
3. 支持 pause/resume/retry/cancel/clearFinished。
4. 并发请求使用 abort/latest-wins 或 request sequence，避免旧响应覆盖新状态。
5. 提供 active/failed/completed 计数与稳定排序。
6. 未知 kind/status 必须安全降级，不得使整个页面崩溃。

### S2-C Global Task Center UI and navigation

Owned files:
- `src/lib/features/jobs/JobsPanel.svelte`

- 如需要，`src/lib/components/CommandDrawer.svelte`

2. 页面包含摘要、过滤、任务列表、空态、加载态、错误态。

### S2-D Download compatibility integration

Owned files:
- `src/lib/components/DownloadPage.svelte`
- 与下载投影直接相关的前端测试。

Requirements:
1. DownloadPage 继续展示下载专用信息（速度、ETA、文件大小）。
2. 控制动作改为复用统一 jobs store/capabilities。
3. 保留 legacy 下载列表 fallback，直到后端迁移完成。
4. 避免同一任务在 persistent 和 legacy list 中重复。
5. 不改变 anime download 的现有独立能力，除非已有统一 ID 可可靠映射。

## 5. UX States

- Empty: 暂无任务，并解释任务会在下载、导入、刮削等操作后出现。
- Loading: 首次 skeleton；后台刷新不清空已有内容。
- Error: 保留旧数据并显示可重试 banner。
- Recovered: 显示“从上次会话恢复”。
- Failed: 显示脱敏 error kind/message 和可用修复动作。
- Active: 显示 progress；未知总量任务使用 indeterminate UI。

Manual Tauri matrix:

2. 暂停、继续、取消、失败重试。
3. 运行任务时退出应用，重启后显示 recovered 状态。
4. 清理完成任务不影响 active jobs。
5. 后端不可用时页面保留最后快照并允许重试。

## 7. Definition of Done

- 全局 Task Center 可从主导航或工具抽屉进入。
- 通用任务和下载任务共享控制面与状态语义。
- 后端恢复、过滤、控制命令有 Rust contract tests。
- 前端 store 有并发、失败回滚、过滤和 capability tests。
- UI 有键盘/可访问性/空态/错误态验证。
- DownloadPage 无功能倒退。
