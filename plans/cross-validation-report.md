## 三、子代理分工策略

**范围隔离（核心原则）**：每个子代理拥有**互不相交的文件集合**，从源头避免并发写冲突。

- **Agent A（清理）**：仅 `.playwright-mcp/`、` .gitignore`
- **Agent B（游戏库 QA）**：仅 `src/lib/stores/gameLibrary.svelte.ts` + 测试，其余只读
- **Agent 2（播放器）**：仅 `src-tauri/src/video_extractor.rs`、`video_proxy.rs`
- **Agent 3（迁移移除）**：迁移相关文件（已删除/修改清单）
- **主代理（集成敏感文件）**：`App.svelte`、`SwitchHome.svelte`、`GameCard.svelte`、`gameLibrary` store 入口——这些文件横跨多个 spec，集中由主代理改动以保证 UI 一致性

**并行通道**：清理 / 游戏库 / 播放器 / 迁移移除 四条独立泳道，互不阻塞。

**第二人复核**：播放器修复先由 Agent 2 完成，再由独立代理 Agent C 重新走查其 diff，实现"第二人评审"式交叉验证（避免作者盲区）。

**集成闸门集权**：所有子代理回报后，由 D 统一跑全量门禁——子代理各自只验证自己局部，全局一致性必须集中校验。

## 四、交叉验证机制

1. **Rust 编译闸门 — `cargo check`**
   tauri-build 在编译期解析 `capabilities/*.json` 中的权限标识符，缺失即报错。专门防"删了命令但 capability 还引用"类问题（本次 S8 正是栽在这里）。
2. **前端类型闸门 — `svelte-check`（`npm run check`）**
   防 `invoke` 参数 / 类型不匹配、Svelte 5 runes 误用。
3. **悬空引用扫描**
   以"被移除符号"为种子（`migrateFromCsharp` / `verifyMigration` / `MigrationPage` / `csharp_migration`），全仓 Grep 其所有引用，**排除** `target/`、`dist/`、`gen/`、`node_modules/` 等生成物后人工判定。
4. **capability↔command 一致性**
   tauri-build 通过即证明 `capabilities/*.json` 中每个权限标识符都能解析到已注册命令。
5. **契约对等（前端↔Rust）**
   Agent B 逐一比对 `invoke("xxx")` 与 `#[tauri::command]` 命名（camelCase→snake_case）是否一致。
6. **模块重校验**
   播放器由独立代理 Agent C 重新走查 Agent 2 的改动。
7. **单元 + 手动测试矩阵**
   每个核心模块覆盖正常 / 边界 / 异常三类场景。

**正常流程**
- 打开全部游戏库：赛博朋克黑绿渐变底色正确渲染，封面不透过 `.sw-bg`

**边界条件**
- 空游戏库：显示空状态 + 「添加本地游戏」入口

- 大量游戏（虚拟滚动）：视图切换流畅，无选择栏长短闪烁

**异常场景**
- 用户取消添加对话框：不再误报"游戏库加载失败：已取消"（S7 修复：取消正则 `/已取消|cancelled|cancel/i` 识别后静默返回）
- 导入过程网络/IO 异常：错误提示明确，不污染全局 banner
- 数据库读取失败：降级提示，不崩溃

**单元覆盖**：`gameLibrary.test.ts` 9/9、`gameSelection.test.ts` 7/7、`router.test.ts` 11/11、`continue.test.ts` 24/24、`game.test.ts` 8/8 等，合计 **107/107 通过**。

## 六、播放器模块验证矩阵（S5 / S6）

**正常流程**
- 打开番剧 → headless WebView 加载规则 JS → 提取真实视频 URL
- 代理重写 m3u8 分片地址 → hls.js 正常起播，画面无黑屏
- 广告 URL 出现在资源请求中：被 `!is_ad_url` 过滤，不污染结果通道

**边界条件**
- CDN 返回的 HLS manifest 无 `.m3u8` 扩展名：`video_proxy` 通过 `#EXTM3U/#EXT-X-` 字节嗅探识别并重写相对分片 → 不再黑屏
- manifest Content-Type 异常（非 `application/vnd.apple.mpegurl`）：嗅探兜底仍生效
- manifest 前导含空白 / BOM：嗅探忽略前导
- 头部 8KB 读取后正确**先发送 head 字节再续传 body**，避免缓冲饥饿卡死

**异常场景**
- 规则加载失败：watchdog 超时兜底，提示用户
- 代理连接断开：原生 video 元素降级 + 错误提示
- 广告 / 追踪域名请求：被 `is_ad_url` 拦截，结果通道稳定
- 分片 403 / 过期：hls.js 原生重试

> 注：headless GUI 环境无法跑 Tauri E2E，上述以"静态门禁 + 代码走查 + 手动测试矩阵"交付；真机起播验证建议留在手动回归。
