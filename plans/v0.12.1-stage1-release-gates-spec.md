# v0.12.1 Stage 1 — Release Gate Closure SPEC

## 1. Objective

将当前 `v0.12.1` 从“已打标签但仍处于开发/发布阻塞状态”收口为可重复验证的发布候选。范围仅包括质量门、构建/测试可复现性、更新器签名发布策略和发布证据，不引入新产品功能。

## 2. Current Baseline (2026-07-11)

- `master` 工作树干净，HEAD 为 `cc71496`。
- `npm run verify:commands` 通过：328 条 Tauri 命令契约一致。
- `npm run verify:versions` 通过：版本均为 `0.12.1`。
- `cargo fmt --check` 失败：`src-tauri/src/lib.rs` 模块声明顺序不符合 rustfmt。
- `cargo clippy --all-targets --all-features -- -D warnings` 失败：`Downloader` import 与 `download_dir` 未使用。
- 当前 Codex 环境中 `svelte-check`/Vitest 加载 Vite 配置时出现 Windows 路径访问错误；必须区分仓库缺陷与执行环境限制。
- Release workflow 当前 `includeUpdaterJson: false`，更新器产物在签名密钥修复前被主动禁用。

## 3. In Scope

### S1-A Rust gate closure

Owned files:
- `src-tauri/src/lib.rs`
- 如测试证明必需，可修改与该文件初始化逻辑直接相关的 Rust 测试；不得重构业务模块。

Requirements:
1. `cargo fmt --check` 通过。
2. `cargo clippy --all-targets --all-features -- -D warnings` 通过。
3. 未使用的默认下载目录必须删除或恢复真实用途，不允许仅用 `#[allow]` 压制。
4. 不改变现有命令注册数量和应用状态初始化语义。

### S1-B Frontend validation reproducibility

Owned files:
- `scripts/test-unit.mjs`
- `vite.config.ts`
- `vitest.config.ts`
- `svelte.config.js`
- 可新增 `scripts/` 下的最小验证 fixture/test。

Requirements:
1. 调查 Windows drive normalization、shell spawning、config resolution。
2. 不得通过扩大磁盘扫描范围或关闭测试来规避错误。
3. 保持 `npm run check`、`npm run test:unit`、`npm run build` 的现有 CLI 契约。
4. 若错误仅由受限沙箱造成，应留下可机读诊断，而不是引入无效代码改动。

### S1-C Updater/release readiness

Owned files:
- `.github/workflows/release.yml`
- `.github/workflows/ci.yml`
- `scripts/verify-updater-artifacts.mjs`
- `scripts/verify-release-artifacts.ps1`
- 可新增 `scripts/` 下更新器验证测试和发布说明。

Requirements:
1. 不可在仓库中写入私钥或真实 secret。
2. 明确“签名 secret 可用”和“不可用”两种 CI 行为。
3. 更新器未启用时，发布流程必须明确标记为阻塞/降级，不得伪装为完整自动更新发布。
4. 更新器启用时必须校验 `latest.json`、HTTPS URL、签名、版本一致性。
5. 保留 draft release + installed upgrade/rollback approval 的现有发布政策。

## 4. Out of Scope

- 新 Provider、新媒体源。
- Task Center 产品功能。
- UI 大改。
- 真实签名密钥轮换和 GitHub Secret 写入（需仓库管理员操作）。

## 5. Verification Matrix

```powershell
git diff --check
npm run verify:versions
npm run verify:commands
npm run test:command-contract
cargo fmt --manifest-path src-tauri/Cargo.toml -- --check
cargo clippy --manifest-path src-tauri/Cargo.toml --all-targets --all-features -- -D warnings
cargo test --manifest-path src-tauri/Cargo.toml --all-targets --all-features
npm run check
npm run test:unit
npm run build
npm run test:bundle-budget
npm run verify:bundle-budget
npm run test:release-evidence
npm run verify:updater
```

`verify:updater` 只有在存在签名 updater artifact 时应通过；无 artifact 的普通开发工作区应给出明确的“未生成”结果，Release job 则必须失败或保持 draft 阻塞。

## 6. Definition of Done

- Rust fmt/clippy/test 全绿。
- Frontend check/unit/build 在普通 Windows 开发环境和 CI 可重复运行。
- updater 发布路径有明确的 secret gate、产物校验和失败语义。
- 所有改动有自动测试或可执行验证命令。
- 无业务功能回归、无命令契约漂移、无 secret 泄露。
