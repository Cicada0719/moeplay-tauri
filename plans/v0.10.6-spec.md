# v0.10.6 详细规格：ContinueHub Dashboard

## 1. 数据模型

### 1.1 ContinueItem（增强）

```ts
export interface ContinueItem {
  id: string;                  // 全局唯一，格式 `${type}-${rawId}`
  type: "game" | "anime" | "comic";
  title: string;
  cover: string | null;
  progress: number;            // 0-100
  progressLabel: string;       // 如 "12.5h" / "第3话" / "42%"
  lastActivity: number;        // 时间戳 ms
  subtitle?: string;           // 如开发商、番剧来源、漫画作者
  actionLabel?: string;        // 如 "继续游玩" / "继续观看" / "继续阅读"
}
```

### 1.2 ContinueStats

```ts
export interface ContinueStats {
  totalCount: number;          // 进行中总数
  gameCount: number;
  animeCount: number;
  comicCount: number;
  todayMinutes: number;        // 今日游戏 + 番剧/漫画估算分钟数
  weekMinutes: number;         // 近 7 天
  streakDays: number;          // 最近连续有活动的天数
}
```

## 2. 纯函数规格（src/lib/utils/continue.ts）

### 2.1 游戏进度

```ts
export function computeGameProgress(game: GameLike): number
```

- 若 `achievements_total > 0`：`achievements_unlocked / achievements_total * 100`。
- 否则按状态映射：
  - `not_started` / `plan_to_play` → 0
  - `playing` / `replaying` → 15
  - `on_hold` → 30
  - `completed` / `dropped` → 100

### 2.2 游戏副标题与 Action

```ts
export function gameSubtitle(game: GameLike): string
```

- 优先返回 `developerOf(game)`，否则 `platformOf(game)`，否则空字符串。

```ts
export function gameActionLabel(status: string): string
```

- `not_started` → "开始游玩"
- `playing` → "继续游玩"
- `completed` → "已通关"
- `on_hold` → "继续游玩"
- `plan_to_play` → "开始游玩"
- 其他 → "打开"

### 2.3 时长聚合

```ts
export function aggregateTodaySeconds(sessions: PlaySession[], now?: Date): number
export function aggregateWeekSeconds(sessions: PlaySession[], now?: Date): number
```

- `today`：从 `now` 当天 00:00:00 到 `now` 之间所有 `duration_seconds` 之和。
- `week`：从 `now` 往前推 6 天 00:00:00 到 `now` 之间所有 `duration_seconds` 之和。
- 只统计 `duration_seconds > 0` 的会话。

### 2.4 Streak

```ts
export function computeStreakDays(activityTimestamps: number[], now?: Date): number
```

- 统计从今天往前连续有活动的天数。
- 今天有活动则从今天算起；今天无活动则从昨天往前算。
- 返回连续天数（>=0）。

### 2.5 Item 构建

```ts
export function buildGameContinueItem(game: Game): ContinueItem
export function buildAnimeContinueItem(h: AnimeHistory): ContinueItem
export function buildComicContinueItem(h: ReadRecord): ContinueItem
export function buildContinueItems(
  games: Game[],
  animeHistory: AnimeHistory[],
  comicHistory: ReadRecord[]
): ContinueItem[]
```

- 过滤已完成/已弃坑的游戏；番剧要求 `lastEpisode > 0`；漫画无额外过滤。
- 按 `priorityScore` 降序排列，取前 30 条。

### 2.6 优先级评分

```ts
export function priorityScore(item: ContinueItem): number
```

- 基础分 = `lastActivity` 时间戳（毫秒） * 0.0000001（保证时间近优先）。
- 类型加权：
  - 游戏中 `playing` 状态额外 + 200
  - 番剧 + 50
  - 漫画 + 50
- 进度加权：进度 1-99 额外 + 100（做到一半的最应该继续）。

### 2.7 Stats 构建

```ts
export function buildContinueStats(
  items: ContinueItem[],
  games: Game[],
  animeHistory: AnimeHistory[],
  comicHistory: ReadRecord[]
): ContinueStats
```

- counts 来自 `items`。
- 今日/本周分钟：
  - 游戏：所有进行中游戏的 sessions 聚合 / 60。
  - 番剧：以 `lastEpisode * 24` 粗略估算，只统计今天/本周有更新的条目。
  - 漫画：以 `last_order * 2` 粗略估算，只统计今天/本周有更新的条目。
- streak：收集所有 items 的 `lastActivity` 时间戳 + 所有游戏 session 的起始日期，计算连续天数。

## 3. Store 改造（src/lib/stores/continue.svelte.ts）

- 内部使用 `$state` 保存 `_items`。
- `$effect` 监听 `gameStore.allGames`、`animeStore.history`、`comicStore.readHistory` 变化并重新计算。
- 导出：
  - `items: ContinueItem[]`
  - `games / anime / comics: ContinueItem[]`
  - `stats: ContinueStats`
  - `topItem: ContinueItem | null`
  - `totalCount: number`

## 4. UI 规格

```
┌─────────────────────────────────────────────────────────────┐
│ 今日中枢                              进行中 {totalCount}   │
├─────────────────────────────────────────────────────────────┤
│ [今日 {todayMinutes}m] [本周 {weekMinutes}m] [连续 {streakDays}天] [游戏 {n}] [番剧 {n}] [漫画 {n}] │
├─────────────────────────────────────────────────────────────┤
│ 继续                                                          │
│ ┌─────────────────────────────────────────────────────────┐ │
│ │ [封面]  标题                                             │ │
│ │         副标题 · 进度 60%                                │ │
│ │         [继续游玩]                                       │ │
│ └─────────────────────────────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────┘
```

### 4.2 ContinueCard

- 左侧 44×59 封面。
- 中间：标题、副标题、进度条（当 `progress > 0` 时显示）、进度标签。
- 右侧：操作图标（播放/书本/手柄）。
- 悬停显示 action 提示。

### 4.3 空状态

- 标题：「还没有进行中内容」
- 描述：「导入本地游戏、安装番剧规则源或登录哔咔，这里会自动聚合。」
- 按钮：「导入游戏」「去追番」「去看漫」。

## 5. 测试规格

`src/lib/utils/continue.test.ts` 至少覆盖：

1. `computeGameProgress`：成就、状态映射、边界值。
2. `aggregateTodaySeconds` / `aggregateWeekSeconds`：跨天、空数组、无效 duration。
3. `computeStreakDays`：连续、断档、今天无活动。
4. `buildContinueItems`：过滤已完成游戏、排序、截断。
5. `priorityScore`：时间、进度、类型加权。
6. `buildContinueStats`：counts、分钟聚合、streak。
