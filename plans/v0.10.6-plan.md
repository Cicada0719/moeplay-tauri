# v0.10.6 优化计划：把 ContinueHub 升级为「今日中枢 Dashboard」

> 版本：0.10.5 → 0.10.6  
> 目标：让原本存在感较弱的「继续」页变成每一个展示栏位都有实际作用的今日中枢，同时补齐前端数据层与单元测试。

## 背景

当前 `ContinueHub` 只是把最近的游戏/番剧/漫画按时间排序列出，信息密度低、操作入口单一、空状态无引导。本轮优化的核心是把该页面升级为 Dashboard：

- 顶部展示今日/本周活跃度、连续活跃天数等数据，让用户一眼看到「今天玩了多久」。
- 智能排序「最该继续」的内容，而非纯时间倒序。
- 每个卡片展示真实进度条、副标题、操作意图。
- 空状态提供一键跳转入口，不再只是提示文字。
- 数据层拆出可测试的纯函数，降低 `continue.svelte.ts` 的复杂度。

## 本轮整合的 GitHub 优秀项目

- [date-fns/date-fns](https://github.com/date-fns/date-fns) — 替换手写 `timeAgo` 与日期聚合逻辑，提供一致、可国际化的日期/时间处理。

## 改动范围

| 类别 | 文件/模块 | 说明 |
|------|-----------|------|
| 新增依赖 | `package.json` | `date-fns` |
| 新增工具模块 | `src/lib/utils/continue.ts` | 进度计算、时长聚合、streak、item 构建、优先级评分 |
| 新增测试 | `src/lib/utils/continue.test.ts` | 覆盖上述纯函数 |
| 重构 Store | `src/lib/stores/continue.svelte.ts` | 使用纯函数生成 items 与 stats，暴露更丰富的派生状态 |
| UI 升级 | `src/lib/components/ContinueHub.svelte` | Dashboard 布局、Hero 统计、Top Action、分类网格 |
| UI 升级 | `src/lib/components/ContinueCard.svelte` | 进度条、副标题、明确 action 图标 |
| 版本号 | `package.json` / `Cargo.toml` / `tauri.conf.json` / `CHANGELOG.md` | 0.10.5 → 0.10.6 |

## 验收标准

1. `npm run test:unit` 通过，新增测试覆盖率完整。
2. `npm run check` 无类型错误。
3. ContinueHub 能正确展示：总进行中数、今日分钟、本周分钟、连续活跃天数、各类型数量。
4. 每个 ContinueCard 显示进度条（游戏按成就/状态估算，番剧/漫画展示集数）。
5. 空状态提供「添加游戏」「去追番」「去看漫」的快捷入口。
6. 版本号在所有配置文件中一致为 0.10.6。

## 下一轮的入口

本轮完成后，将继续扫描可优化点，候选方向：

- 番剧播放器整合 `hls.js` 之外的现代化播放器 UI（如基于 [nplayer](https://github.com/woopen/nplayer) 或 Video.js 的自定义皮肤）。
- 统计页接入图表库（如 [Unovis](https://github.com/f5/unovis) 或 [LayerChart](https://github.com/techniq/layerchart)）。
- 设置页按领域分组并增加搜索/快捷键提示。
