# v0.10.11 优化计划：URL hash 路由持久化

> 版本：0.10.10 → 0.10.11  
> 目标：把当前应用内的视图状态同步到地址栏 hash，让刷新、前进/后退、外部链接都能恢复到原来的视图与选中的游戏。

## 背景

目前 `uiStore.currentView` 与 `gameStore.selectedId` 只保存在内存中，窗口刷新或应用重启后总是回到首页。桌面端用户期望“关闭再打开还能回到刚才在看的位置”。本轮引入轻量级 hash 路由：不引入外部路由库，仅通过 `window.location.hash` 实现状态持久化，保持现有组件架构不变。

## 改动范围

| 类别 | 文件/模块 | 说明 |
|------|-----------|------|
| 新增 Store | `src/lib/stores/router.svelte.ts` | hash 解析、生效、双向同步、`popstate`/`hashchange` 监听 |
| 新增测试 | `src/lib/stores/router.test.ts` | parseHash / buildHash / 视图合法性校验 |
| 更新 Store | `src/lib/stores/ui.svelte.ts` | 移除 `?view=` 查询参数初始化，默认回到 `home` |
| 更新 Store | `src/lib/stores/games.svelte.ts` | 导出 `selectedId` setter 或保持 `selectGame` 接口 |
| 更新入口 | `src/App.svelte` | 启动后调用 `router.init()`，处理 hash 恢复 |
| 版本号 | 配置文件 + CHANGELOG | 0.10.10 → 0.10.11 |

## 验收标准

1. `npm run test:unit` 通过（含新增 router 测试）。
2. `npm run check` 无新增错误。
3. `npm run build` 成功。
4. 切换视图时地址栏 hash 同步变化（如 `#home`、`#settings`、`#stats`、`#game-detail?id=xxx`）。
5. 刷新页面后恢复到最后一个合法视图；游戏详情恢复选中游戏。
6. 浏览器/嵌入式 WebView 的前进、后退按钮能触发视图回退。
7. 非法 hash 自动回退到 `home`，不抛错。
8. 版本号统一为 0.10.11。
