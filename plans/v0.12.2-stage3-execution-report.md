# Stage 3 / v0.12.2 交付报告：任务生产者与可观测性

## 已交付

- **版本与存储**：应用版本更新为 `0.12.2`；SQLite schema 升级至 v6，新增 `background_job_events`。事件按任务顺序持久化，单任务最多保留 200 条，终态任务按 30 天策略清理；URL 凭据、Token、请求头和敏感字段会在持久化前脱敏。
- **受控任务契约**：后端引入版本化 `JobOperation`（当前 envelope v1）和启动时固定 dispatcher。持久化操作仅包含实体/Provider 标识及非敏感引用；队列仅自动重派无运行时输入的 `UpdateCheck`，其余恢复时不能安全重放的任务显式失败并记录 `process_restart`，绝不静默重复执行。
- **生产者接入**：导入（含 Steam）、普通/合并刮削、Anime/Comic 来源验证、备份、恢复、诊断导出和更新检查均创建后端拥有的任务并写入真实状态/事件。
- **生命周期与恢复**：任务具有事件时间线、取消句柄和单调终态保护；取消后的迟到成功不能覆盖 `cancelled`。应用重启时下载运行任务转为 `paused`，不能安全重放的运行或排队操作标记为 `failed` 且记录 `process_restart`；`UpdateCheck` 由后端 dispatcher 重派。
- **Task Center UI**：新增任务详情抽屉、事件时间线、来源跳转上下文、脱敏错误复制/恢复入口和全局活动徽标；任务列表保留既有下载专属暂停/恢复边界。
- **公开命令**：新增 `get_task_detail`、`get_task_events`、`start_update_check_task`，并已写入 Tauri 处理器、build-time command contract、capability 和生成权限。更新检查在 Tauri 后端调用签名 updater；前端只收到 `available` 和任务投影。
