# v0.10.10 优化计划：主题系统扩展

> 版本：0.10.9 → 0.10.10  
> 目标：在现有「深色 / 浅色 / 樱夜」基础上，增加「跟随系统 / 纯黑 / 高对比」三种主题，并通过集中式主题工具解决 `system` 模式的监听与回退问题。

## 背景

当前主题写死在 `settingsStore` 中，仅支持 dark / light / sakura，且 sakura 实际上没有独立 CSS 变量块（回退到 dark）。随着应用在 Windows 桌面与不同显示设备上运行，用户对 OLED 省电、高对比可访问性、跟随系统明暗的需求越来越明显。本轮将主题逻辑抽取为可测试的纯工具模块，新增三种主题并完善设置页入口。

## 改动范围

| 类别 | 文件/模块 | 说明 |
|------|-----------|------|
| 新增工具 | `src/lib/utils/theme.ts` | 主题类型、合法性校验、`system` 解析、应用与监听器管理 |
| 新增测试 | `src/lib/utils/theme.test.ts` | `resolveTheme`、`isValidTheme`、`applyTheme` 行为覆盖 |
| 更新 Store | `src/lib/stores/settings.svelte.ts` | 使用 `theme.ts` 初始化与持久化，移除硬编码 localStorage 逻辑 |
| 更新组件 | `src/lib/components/SettingsPage.svelte` | 主题选择器使用 `APP_THEMES`，展示 6 个主题选项 |
| 更新图标 | `src/lib/components/Icon.svelte` | 新增 `moon`、`contrast` 图标 |
| 更新样式 | `src/app.css` | 新增 `[data-theme="black"]`、`[data-theme="contrast"]` 变量块 |
| 版本号 | 配置文件 + CHANGELOG | 0.10.9 → 0.10.10 |

## 验收标准

1. `npm run test:unit` 通过（含新增 theme 测试）。
2. `npm run check` 无新增错误。
3. `npm run build` 成功。
4. 设置页可切换 6 种主题：深色、浅色、樱夜、跟随系统、纯黑、高对比。
5. 「跟随系统」在系统主题变化时自动切换为当前生效的深色或浅色，重启后仍记住偏好。
6. 纯黑与高对比主题在主要页面（游戏库、统计、设置）视觉可用，边框与文字对比度足够。
7. 版本号统一为 0.10.10。
